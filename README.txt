 //Wells Analytics — аналитика данных нефтяных скважин

Проект аналитики данных телеметрии нефтедобывающих скважин на основе 
данных ПЛК Siemens (SCADA WinCC OA, Modbus/S7+).

 //Структура проекта

- `sql/` — схема PostgreSQL, тестовые данные, аналитические VIEW 
  (оконные функции, CTE, back allocation)
- `powerbi/` — интерактивные дашборды (добыча, простои, электрика, 
  back allocation)
- `excel/` — прототипы аналитики (back allocation, дашборд АГЗУ, 
  сводная аналитика АСУ ТП)
- `airflow/` — DAG для проверки доступности источника данных, 
  свежести данных и email-алертов
- `docs/` — техническая документация и данные из скважин

 //Стек технологий

PostgreSQL · Power BI · Apache Airflow · Docker · Excel

 //Ключевые задачи, решённые в проекте

- Тренды добычи и обводнённости с автоматическими флагами отклонений
- Back allocation — распределение фактической добычи по скважинам
- Парето-анализ причин простоев оборудования
- Мониторинг электрических параметров (cos φ, дисбаланс токов/напряжений)
- Автоматизированный пайплайн с проверкой качества данных и алертами
 
 //Моделирование бизнес-процесса (BPMN)

Диаграмма AS-IS / TO-BE процесса обнаружения и реагирования на остановку 
скважины — обоснование ценности автоматизированного мониторинга:

![BPMN диаграмма мониторинга остановок](docs/bpmn_well_monitoring.png)

| Метрика | AS-IS | TO-BE |
|---|---|---|
| Время обнаружения простоя | Не гарантировано, зависит от оператора | ≤15 минут (интервал DAG) |
| Требуется присутствие человека | Да | Нет |
| Автоматическое уведомление | Нет | Да, email |

Полная версия: [PDF](docs/bpmn_well_monitoring_as-is_to-be.pdf)

 //Примечание

Тестовые данные в проекте синтетические, структурированы по образцу 
реальных тегов SCADA WinCC OA. Прямая интеграция с рабочим сервером 
SCADA в процессе реализации.
